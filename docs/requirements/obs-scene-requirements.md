# Требования: ручное управление сценами OBS

## 1. Цель

Расширить существующее приложение возможностью **ручного управления
сценами OBS** через:

-   Telegram‑бот
-   Web‑интерфейс (SSR)

Сцены OBS в системе представляют собой **способы подключения мастера к
игровому столу**.

⚠️ Автоматическое переключение сцен **не входит в текущую версию
требований**.\
Все переключения выполняются **только вручную**.

------------------------------------------------------------------------

## 2. Контекст системы

В системе уже существуют:

-   backend на Node.js
-   Telegram‑бот
-   Web server
-   server-side rendering
-   интеграция с OBS

OBS устанавливается и **настраивается вручную**.

Приложение:

-   не создаёт сцены
-   не конфигурирует источники
-   **только переключает заранее настроенные сцены**

------------------------------------------------------------------------

## 3. Использование сцен OBS

Сцены OBS используются как **режимы подключения мастера**.

Примеры:

  Сцена     Назначение
  --------- --------------------------------------
  chrome    подключение через браузер Chrome
  capture   подключение через HDMI карту захвата
  ndi       подключение через NDI
  backup    резервная сцена
  default   сцена безопасного состояния

Все сцены:

-   создаются вручную в OBS
-   настраиваются вручную в OBS
-   в приложении используются **только по имени**

------------------------------------------------------------------------

## 4. Специальные сцены

### 4.1 Backup сцена

Имя сцены:

    backup

Назначение:

резервный режим подключения.

Требования:

-   пользователь может переключиться на `backup`
-   пользователь может переключиться с `backup` на любую другую сцену
-   интерфейс должен показывать, что активна сцена `backup`

------------------------------------------------------------------------

### 4.2 Default сцена

Имя сцены:

    default

Назначение:

безопасное состояние системы.

Требования:

-   пользователь может переключиться на `default`
-   интерфейс должен отображать активность `default`

------------------------------------------------------------------------

## 5. Конфигурация сцен

Приложение **само не создаёт сцены**, а только использует уже настроенные в OBS.

**Источник истины — OBS.** Список сцен и проверка существования сцены берутся из OBS. Если конфигурационного файла сцен нет (`SCENES_CONFIG_PATH` не задан или файл отсутствует) или названия в конфиге не совпадают со сценами в OBS — это не ошибка: приложение **полностью полагается на настройки OBS**.

Опционально может использоваться файл конфигурации (например для отображаемых подписей или типов сцен в UI). Пример формата:

``` json
[
  { "name": "main",    "title": "Main projector", "type": "main",    "enabled": true },
  { "name": "chrome",  "title": "Chrome",         "type": "output",  "enabled": true },
  { "name": "capture", "title": "HDMI Capture",   "type": "output",  "enabled": true },
  { "name": "backup",  "title": "Backup",         "type": "backup",  "enabled": true },
  { "name": "default", "title": "Default",        "type": "default", "enabled": true }
]
```

При отсутствии конфига список сцен и переключение выполняются по данным из OBS.

Допустимые значения поля `type`:

- `main` — проекторная сцена (не переключается пользователем, держится на проекторе);
- `output` — агрегирующая/управляющая сцена, доступна для переключения;
- `input` — входная сцена, также доступна для переключения;
- `backup` / `default` — специальные безопасные сцены.

Для целей UI и Telegram‑бота **переключаемыми** считаются сцены:

- без заданного `type` (обратная совместимость со старыми конфигами);
- с `type` отличным от `main` и при этом `enabled !== false`.

Сцены с `type: "main"` и/или `enabled: false` из списка для переключения исключаются.

Имя проекторной сцены задаётся переменной окружения:

```text
OBS_PROJECTOR_SCENE_NAME=main
```

Если `OBS_PROJECTOR_SCENE_NAME` не задана или сцена с таким именем отсутствует в OBS, модуль проекторной сцены:

- пытается использовать первую сцену с именем, начинающимся на `output.` (fallback);
- при неудаче пишет предупреждение в лог и **не запускает projector**, чтобы избежать неожиданного вывода.

------------------------------------------------------------------------

## 6. Управление сценами

Система должна позволять:

-   получить список сцен
-   узнать текущую сцену
-   переключить сцену

Все действия выполняются **только по команде пользователя**.

------------------------------------------------------------------------

## 7. Telegram команды

### `/scenes`

Показать список сцен.

------------------------------------------------------------------------

### `/scene <name>`

Переключить сцену OBS.

Пример:

    /scene chrome

------------------------------------------------------------------------

### `/current`

Показать текущую сцену OBS.

------------------------------------------------------------------------

### `/backup`

Переключить систему на сцену `backup`.

------------------------------------------------------------------------

### `/default`

Переключить систему на сцену `default`.

------------------------------------------------------------------------

### `/status`

Показать статус системы.

Пример:

    OBS: connected
    Current scene: chrome

------------------------------------------------------------------------

## 8. Web интерфейс

Маршрут страницы:

    GET /obs/scenes

Страница должна отображать:

-   статус подключения к OBS
-   текущую сцену
-   список доступных сцен
-   кнопки переключения сцен

Дополнительные кнопки:

-   Backup
-   Default

Активная сцена должна быть **визуально выделена**.

------------------------------------------------------------------------

## 9. Web API

### Переключение сцены

    POST /obs/scene

Body (только JSON, `Content-Type: application/json`):

    {"scene": "<name>"}

Пример: `{"scene": "chrome"}`. Формат `application/x-www-form-urlencoded` не поддерживается.

------------------------------------------------------------------------

### Backup сцена

    POST /obs/scene/backup

------------------------------------------------------------------------

### Default сцена

    POST /obs/scene/default

------------------------------------------------------------------------

## 10. Интеграция с OBS

Приложение должно подключаться к OBS через **OBS WebSocket API**.

**Уточнение:** если бы существовал способ управлять OBS напрямую (без WebSocket), следовало бы использовать его; на практике управление сценами OBS доступно через OBS WebSocket API.

Функции:

-   установка соединения
-   проверка состояния соединения
-   переключение сцен

------------------------------------------------------------------------

## 11. Обработка ошибок

Система должна корректно обрабатывать:

-   OBS недоступен
-   WebSocket соединение разорвано
-   сцена отсутствует в OBS (запрос переключения на несуществующую сцену)
-   неавторизованный Telegram пользователь
-   неавторизованный web пользователь

Все ошибки должны возвращать **понятные сообщения**.

------------------------------------------------------------------------

## 12. Авторизация

### Telegram

Команды разрешены **только whitelisted пользователям**.

------------------------------------------------------------------------

### Web

Web интерфейс должен быть защищён.

Возможные варианты:

-   basic auth
-   session auth
-   reverse proxy auth
-   доступ только из локальной сети

**Уточнение:** на данном этапе защита Web **не требуется** — можно не реализовывать.

------------------------------------------------------------------------

## 13. Логирование

Система должна логировать:

-   запуск приложения
-   подключение к OBS
-   восстановление соединения
-   переключение сцен
-   Telegram команды
-   web запросы
-   ошибки
-   попытки неавторизованного доступа

Пример (формат плоский, key=value, для удобства grep):

    scene_switch source=telegram from=chrome to=capture success=true
    scene_switch source=web from=capture to=backup success=true
    obs_connection status=reconnected

------------------------------------------------------------------------

## 14. Конфигурация окружения

Пример `.env`:

    OBS_HOST=127.0.0.1
    OBS_PORT=4455
    OBS_PASSWORD=...
    ALLOWED_TELEGRAM_USERS=123456789
    SCENES_CONFIG_PATH=./config/scenes.json

Переменная `SCENES_CONFIG_PATH` **опциональна**: при её отсутствии список сцен и переключение выполняются по данным из OBS.

------------------------------------------------------------------------

## 15. Минимальный scope реализации

Первая версия должна включать:

-   сервис управления сценами OBS (список и текущая сцена из OBS, переключение по имени)
-   Telegram команды
-   SSR страницу управления сценами
-   Web API для переключения
-   логирование
-   авторизацию Telegram

------------------------------------------------------------------------

## 16. Критерии готовности

Функциональность считается реализованной если:

-   можно получить список сцен
-   сцены можно переключать через Telegram
-   сцены можно переключать через web интерфейс
-   работает сцена `backup`
-   работает сцена `default`
-   отображается текущая сцена
-   система корректно обрабатывает недоступность OBS
-   неавторизованные пользователи не могут управлять системой

------------------------------------------------------------------------

## 17. Итог

Приложение должно обеспечивать **ручное управление заранее настроенными
сценами OBS**.

Поддерживаемые функции:

-   переключение сцен
-   использование сцены `backup`
-   использование сцены `default`
-   управление через Telegram
-   управление через Web интерфейс

Автоматическое переключение сцен **в текущую версию требований не
входит**.

------------------------------------------------------------------------

## 18. Уточнения (решения по обсуждению)

-   **Защита Web (§12):** на данном этапе не требуется, можно игнорировать.
-   **Управление OBS:** если приложением можно управлять OBS напрямую без WebSocket — управлять напрямую; иначе использовать OBS WebSocket API.
-   **POST /obs/scene:** принимать только JSON (`{"scene": "<name>"}`); `application/x-www-form-urlencoded` не нужен.
-   **Логирование (§13):** формат вывода — плоский (key=value) для grep.
-   **Конфигурация сцен (§5):** приложение не создаёт сцены, только использует. Источник истины — OBS. Если настроек нет (`SCENES_CONFIG_PATH` не задан или файл отсутствует) или названия в конфиге не соответствуют сценам в OBS — не проблема; полностью полагаться на настройки OBS (список сцен и проверка по OBS).
