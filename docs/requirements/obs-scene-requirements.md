# Требования: ручное управление сценами OBS

## 1. Цель

Расширить существующее приложение возможностью **ручного управления
сценами OBS** через:

-   Telegram‑бот
-   Web‑интерфейс (SSR)

Сцены OBS в системе представляют собой **способы подключения мастера к
игровому столу**.

⚠️ Автоматическое переключение сцен **не входит в текущую версию
требований**.\
Все переключения выполняются **только вручную**.

------------------------------------------------------------------------

## 2. Контекст системы

В системе уже существуют:

-   backend на Node.js
-   Telegram‑бот
-   Web server
-   server-side rendering
-   интеграция с OBS

OBS устанавливается и **настраивается вручную**.

Приложение:

-   не создаёт сцены
-   не конфигурирует источники
-   **только переключает заранее настроенные сцены**

------------------------------------------------------------------------

## 3. Использование сцен OBS

Сцены OBS используются как **режимы подключения мастера**.

Примеры:

  Сцена     Назначение
  --------- --------------------------------------
  chrome    подключение через браузер Chrome
  capture   подключение через HDMI карту захвата
  ndi       подключение через NDI
  backup    резервная сцена
  default   сцена безопасного состояния

Все сцены:

-   создаются вручную в OBS
-   настраиваются вручную в OBS
-   в приложении используются **только по имени**

------------------------------------------------------------------------

## 4. Специальные сцены

### 4.1 Backup сцена

Имя сцены:

    backup

Назначение:

резервный режим подключения.

Требования:

-   пользователь может переключиться на `backup`
-   пользователь может переключиться с `backup` на любую другую сцену
-   интерфейс должен показывать, что активна сцена `backup`

------------------------------------------------------------------------

### 4.2 Default сцена

Имя сцены:

    default

Назначение:

безопасное состояние системы.

Требования:

-   пользователь может переключиться на `default`
-   интерфейс должен отображать активность `default`

------------------------------------------------------------------------

## 5. Конфигурация сцен

Список сцен хранится **в конфигурации приложения**.

Приложение **не должно автоматически использовать все сцены из OBS**,
так как часть сцен может быть техническими.

Пример конфигурации:

``` json
[
  { "name": "chrome",  "title": "Chrome",       "type": "working", "enabled": true },
  { "name": "capture", "title": "HDMI Capture", "type": "working", "enabled": true },
  { "name": "backup",  "title": "Backup",       "type": "backup",  "enabled": true },
  { "name": "default", "title": "Default",      "type": "default", "enabled": true }
]
```

------------------------------------------------------------------------

## 6. Управление сценами

Система должна позволять:

-   получить список сцен
-   узнать текущую сцену
-   переключить сцену

Все действия выполняются **только по команде пользователя**.

------------------------------------------------------------------------

## 7. Telegram команды

### `/scenes`

Показать список сцен.

------------------------------------------------------------------------

### `/scene <name>`

Переключить сцену OBS.

Пример:

    /scene chrome

------------------------------------------------------------------------

### `/current`

Показать текущую сцену OBS.

------------------------------------------------------------------------

### `/backup`

Переключить систему на сцену `backup`.

------------------------------------------------------------------------

### `/default`

Переключить систему на сцену `default`.

------------------------------------------------------------------------

### `/status`

Показать статус системы.

Пример:

    OBS: connected
    Current scene: chrome

------------------------------------------------------------------------

## 8. Web интерфейс

Маршрут страницы:

    GET /obs/scenes

Страница должна отображать:

-   статус подключения к OBS
-   текущую сцену
-   список доступных сцен
-   кнопки переключения сцен

Дополнительные кнопки:

-   Backup
-   Default

Активная сцена должна быть **визуально выделена**.

------------------------------------------------------------------------

## 9. Web API

### Переключение сцены

    POST /obs/scene

Body:

    scene=<name>

------------------------------------------------------------------------

### Backup сцена

    POST /obs/scene/backup

------------------------------------------------------------------------

### Default сцена

    POST /obs/scene/default

------------------------------------------------------------------------

## 10. Интеграция с OBS

Приложение должно подключаться к OBS через **OBS WebSocket API**.

Функции:

-   установка соединения
-   проверка состояния соединения
-   переключение сцен

------------------------------------------------------------------------

## 11. Обработка ошибок

Система должна корректно обрабатывать:

-   OBS недоступен
-   WebSocket соединение разорвано
-   сцена отсутствует в конфигурации
-   сцена отсутствует в OBS
-   неавторизованный Telegram пользователь
-   неавторизованный web пользователь

Все ошибки должны возвращать **понятные сообщения**.

------------------------------------------------------------------------

## 12. Авторизация

### Telegram

Команды разрешены **только whitelisted пользователям**.

------------------------------------------------------------------------

### Web

Web интерфейс должен быть защищён.

Возможные варианты:

-   basic auth
-   session auth
-   reverse proxy auth
-   доступ только из локальной сети

------------------------------------------------------------------------

## 13. Логирование

Система должна логировать:

-   запуск приложения
-   подключение к OBS
-   восстановление соединения
-   переключение сцен
-   Telegram команды
-   web запросы
-   ошибки
-   попытки неавторизованного доступа

Пример:

    scene_switch source=telegram from=chrome to=capture success=true
    scene_switch source=web from=capture to=backup success=true
    obs_connection status=reconnected

------------------------------------------------------------------------

## 14. Конфигурация окружения

Пример `.env`:

    OBS_HOST=127.0.0.1
    OBS_PORT=4455
    OBS_PASSWORD=...
    ALLOWED_TELEGRAM_USERS=123456789
    SCENES_CONFIG_PATH=./config/scenes.json

------------------------------------------------------------------------

## 15. Минимальный scope реализации

Первая версия должна включать:

-   конфигурацию сцен
-   сервис управления сценами OBS
-   Telegram команды
-   SSR страницу управления сценами
-   Web API для переключения
-   логирование
-   авторизацию

------------------------------------------------------------------------

## 16. Критерии готовности

Функциональность считается реализованной если:

-   можно получить список сцен
-   сцены можно переключать через Telegram
-   сцены можно переключать через web интерфейс
-   работает сцена `backup`
-   работает сцена `default`
-   отображается текущая сцена
-   система корректно обрабатывает недоступность OBS
-   неавторизованные пользователи не могут управлять системой

------------------------------------------------------------------------

## 17. Итог

Приложение должно обеспечивать **ручное управление заранее настроенными
сценами OBS**.

Поддерживаемые функции:

-   переключение сцен
-   использование сцены `backup`
-   использование сцены `default`
-   управление через Telegram
-   управление через Web интерфейс

Автоматическое переключение сцен **в текущую версию требований не
входит**.
