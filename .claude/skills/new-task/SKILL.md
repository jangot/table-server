# Создать новую задачу из контекста чата

Цель: зафиксировать результаты обсуждения задачи в чате, создать папку задачи и файл `description.md` для дальнейшей работы на этапе `/analyze`.

## Аргументы
- Первый аргумент `$ARGUMENTS[0]` — номер задачи (numberOfPlan). Может быть не передан.
- Остальные аргументы начиная с `$ARGUMENTS[1]` — короткое описание задачи (description). Может быть не передано.

## Определение папки задачи

1. Все артефакты задач находятся в `ai-process/tasks/`
2. Формат имени папки: `ai-process/tasks/{numberOfPlan}-{description}/`
3. Нумерация — трёхзначная с ведущими нулями: 001, 002, ..., 042 и т.д.

**Определение numberOfPlan:**
- Если передан — использовать его
- Если НЕ передан — посмотреть существующие папки в `ai-process/tasks/` и `ai-process/tasks-done/`, взять максимальный номер и прибавить 1

**Определение description (имя папки):**
- Если передано — использовать как часть имени папки (kebab-case, на английском)
- Если НЕ передано — сформировать по контексту обсуждения в чате (kebab-case, на английском)
- Если контекст неоднозначен — спросить у пользователя

**Если папка уже существует:**
- Предупредить пользователя
- Спросить: перезаписать `description.md` или отменить

**Если папки нет** — создать новую.

## Структура выходного файла

Результат: `ai-process/tasks/{numberOfPlan}-{description}/description.md`

Файл описания должен быть **на русском** и содержать:

```markdown
# Задача: {название задачи}

## Суть задачи
{Краткое описание того, что нужно сделать — 3-5 предложений на основе обсуждения в чате}

## Детали и требования
{Перечень конкретных требований, деталей и ограничений, которые были уточнены в ходе обсуждения}

## Контекст
{Дополнительная информация: связанные модули, API, бизнес-логика — если обсуждалось}

## Открытые вопросы
{Вопросы, которые остались невыясненными — если есть}
```

## Порядок работы

1. Проанализировать контекст текущего чата — что обсуждалось, какая задача формировалась
2. Определить номер и описание папки (см. выше)
3. Создать папку задачи
4. Создать `description.md` с зафиксированными деталями из обсуждения
5. Сообщить пользователю:
   - Путь к созданной задаче
   - Краткое резюме того, что зафиксировано
   - Предложить следующий шаг: `/analyze {numberOfPlan}`

## Уточнения
- new-task — нулевой этап конвейера (new-task → analyze → plan → do)
- Не запускать analyze автоматически — только предложить
- Описание должно точно отражать обсуждение, без домыслов
- Если в чате не было достаточно деталей — зафиксировать то, что есть, и указать открытые вопросы
