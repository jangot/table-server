# Выполнить работы по существующему плану

Цель: реализовать задачу строго по плану из plan.md.

## Аргументы
- `$ARGUMENTS` — номер задачи. Может быть не передан.

## Определение задачи

1. Задачи находятся в `ai-process/tasks/`
2. Формат: `ai-process/tasks/{numberOfPlan}-{description}/`
3. Если номер НЕ передан — работаем с последней по номеру задачей

## Предусловия

**Обязательно прочитать перед началом работы:**
1. `plan.md` в папке задачи — основной источник инструкций. Если файла нет — сообщить пользователю и предложить запустить `/plan`
2. `analyze.md` в папке задачи — для понимания контекста и архитектуры
3. Если есть `plan-part2.md` и т.д. — учитывать все части плана

## Порядок работы

### 1. Проверка плана
- Прочитать plan.md целиком
- Если есть неоднозначные требования или несколько вариантов реализации — **сначала задать вопросы** пользователю и получить однозначные ответы, только потом приступать

### 2. Реализация
- Выполнять шаги плана **последовательно**
- Следовать примерам кода из плана
- Использовать сводную таблицу файлов из плана как чеклист

### 3. Проверка после реализации
После завершения всех шагов плана:
- Запустить сборку проекта и убедиться что нет ошибок компиляции: `npm run build`
- Запустить линтер: `npm run lint`
- Запустить тесты: `npm test`

### 4. Перемещение задачи в архив
Если все проверки прошли успешно (сборка, линтер, тесты):
- Переместить папку задачи: `mv ai-process/tasks/{N}-{name} ai-process/tasks-done/`
- Создать папку `ai-process/tasks-done/`, если она не существует
- Сообщить пользователю, что задача перемещена в `tasks-done`

Если проверки не прошли — папку не перемещать, сообщить о проблемах.

## Отслеживание прогресса

Если план большой и может не уместиться в одну сессию:
- Использовать TaskCreate/TaskUpdate для отслеживания шагов
- По завершении сессии — сообщить пользователю какие шаги выполнены, а какие остались

## Уточнения
- do — третий (финальный) этап конвейера (analyze → plan → do)
- Не коммитить изменения автоматически — коммит только по запросу пользователя
- Если в процессе реализации обнаружена ошибка или неточность в плане — сообщить пользователю, не молча отклоняться от плана
