# Пропустить задачу без выполнения

Цель: перенести задачу из `ai-process/tasks/` в `ai-process/tasks-done/` без выполнения, записав причину в `skipped.md` внутри папки задачи.

## Аргументы

- `$ARGUMENTS[0]` — номер задачи. **Обязателен.** Если не передан — сообщить пользователю и остановиться.
- `$ARGUMENTS[1..]` — причина пропуска (остаток строки после номера). **Обязательна.** Если не передана — спросить у пользователя.

## Порядок работы

### 1. Найти папку задачи

- Искать в `ai-process/tasks/` папку с номером `$ARGUMENTS[0]` (формат `{номер}-{description}/`)
- Если папка не найдена — сообщить пользователю и остановиться

### 2. Создать `skipped.md` в папке задачи

Содержимое файла:

```markdown
# Задача пропущена

**Дата:** {сегодняшняя дата в формате YYYY-MM-DD}

## Причина

{текст причины из аргументов}
```

### 3. Переместить папку в `tasks-done`

- Создать `ai-process/tasks-done/`, если не существует
- Переместить: `mv ai-process/tasks/{папка} ai-process/tasks-done/`

### 4. Сообщить пользователю

- Название задачи (папки)
- Путь куда перемещена
- Зафиксированная причина

## Уточнения

- Никаких изменений в коде не делать
- Не запускать сборку, линтер и тесты
- Существующие файлы задачи (description.md, analyze.md, plan.md) не удалять и не изменять
