# Задача: Опциональный конфиг сцен для UI

## Суть задачи

Добавить опциональную поддержку файла конфигурации сцен (`SCENES_CONFIG_PATH`): JSON с полями name, title, type, enabled для обогащения списка сцен при отображении в UI (подписи, типы). Источник истины по наличию и переключению сцен остаётся OBS; при отсутствии конфига или несовпадении имён — не ошибка, работа только по данным из OBS.

## Детали и требования

- **Переменная окружения:** `SCENES_CONFIG_PATH` — опциональная, путь к JSON-файлу. Если не задана или файл отсутствует — список сцен и переключение только по данным OBS.
- **Формат файла (§5 требований):** массив объектов `{ "name", "title", "type", "enabled" }`. Пример: `{ "name": "chrome", "title": "Chrome", "type": "working", "enabled": true }`. Имена в конфиге могут не совпадать со сценами в OBS — тогда такие записи не используются для переключения; список сцен для переключения берётся из OBS.
- **Логика:** сервис сцен (из задачи 012) при наличии конфига обогащает список сцен из OBS: для каждой сцены из OBS подставлять title и type из конфига по полю name; если в конфиге нет записи для имени сцены — использовать только имя из OBS. Поле enabled можно использовать для фильтрации в UI (скрывать сцены с enabled: false), не блокируя переключение по API.
- **Не ошибка:** отсутствие файла, пустой конфиг, несовпадение имён конфига со сценами OBS — приложение не падает, полностью полагается на OBS для списка и переключения.
