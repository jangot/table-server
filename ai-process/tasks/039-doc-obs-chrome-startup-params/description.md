# Задача: Документ — параметры запуска OBS и Chrome для корректной конфигурации

## Суть задачи
Собрать и зафиксировать в одном MD-документе все параметры и настройки **запуска** OBS и Chrome, которые использует текущее POC-приложение для корректной конфигурации этих приложений. Документ нужен для того, чтобы при ручном или внешнем запуске OBS и Chrome можно было задать те же параметры (пути, флаги, переменные окружения и т.д.), что и при запуске из этого приложения — для совместимости с логикой управления, описанной в задачах 037 и 038.

## Детали и требования
- **Результат:** один MD-файл с описанием параметров запуска OBS и Chrome.
- **Содержание документа (что собрать из кодовой базы):**
  - **OBS:**
    - Путь к исполняемому файлу (переменная окружения / конфиг).
    - Аргументы командной строки при запуске (если есть).
    - Переменные окружения, влияющие на запуск или конфигурацию OBS (например, путь к конфигу, профиль, рабочая директория).
    - Требования к состоянию OBS для работы управления (например, включённый WebSocket-сервер, порт, пароль).
  - **Chrome:**
    - Путь к исполняемому файлу (если задаётся).
    - Флаги командной строки (kiosk, remote-debugging-port, autoplay-policy, отключение безопасности, масштаб и т.д.).
    - Переменные окружения, влияющие на запуск Chrome.
    - Требования к состоянию Chrome для работы управления (порт CDP, возможность подключения и т.д.).
- Документ структурированный: отдельные секции для OBS и Chrome, списки параметров с указанием источника (env, конфиг, код). Цель — дать инструкцию «как запустить OBS/Chrome так, чтобы приложение могло ими управлять».
- Не включать в этот документ описание самой логики управления (это в задачах 037 и 038), только параметры запуска и конфигурации, необходимые для корректной работы.

## Контекст
- В новом репозитории OBS и Chrome не будут запускаться из приложения; их нужно будет запускать вручную или другим способом. Чтобы управление работало так же, как в POC, нужна выжимка всех параметров запуска из текущего кода.
