# Задача: Web API для переключения сцен OBS

## Суть задачи

Добавить HTTP API для переключения сцен OBS: POST с именем сцены (только JSON), отдельные эндпоинты для backup и default. Возвращать понятные ошибки при недоступности OBS или несуществующей сцене. Защита Web-интерфейса на данном этапе не требуется (§12 требований).

## Детали и требования

- **POST /obs/scene** (§9 требований): тело только JSON, `Content-Type: application/json`. Формат: `{"scene": "<name>"}`. Пример: `{"scene": "chrome"}`. Формат `application/x-www-form-urlencoded` не поддерживать. При успехе — ответ 200 (или 204). При невалидном теле или отсутствии поля scene — 400 с понятным сообщением. При недоступности OBS или несуществующей сцене — соответствующий код (503/404) и сообщение.
- **POST /obs/scene/backup** — переключить на сцену `backup`. Ответы по той же логике.
- **POST /obs/scene/default** — переключить на сцену `default`. Ответы по той же логике.
- **Логирование:** вызовы API и результат переключения логировать в формате key=value (например source=web, scene=…, success=…).
- **Авторизация Web:** на данном этапе не реализовывать (§12 — защита не требуется). В будущем возможны basic auth, session, reverse proxy или доступ только из локальной сети.
