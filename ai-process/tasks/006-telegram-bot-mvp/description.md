# Задача: Внешний интерфейс — Telegram-бот (MVP)

## Суть задачи

Реализовать приём команд через Telegram Bot API (минимум: команда с URL для открытия). Ввести отдельный модуль пользователей с источником списка разрешённых пользователей из env (с возможностью позже подменить на БД). Проверять авторизацию и вызывать логику переключения URL; неавторизованных отклонять с логированием.

## Детали и требования

- Приём команд через Telegram Bot API (минимум: команда с URL для открытия).
- **Модуль пользователей:** отдельный модуль для работы с разрешёнными пользователями; в MVP источник — переменные окружения (список логинов/usernames или id). Интерфейс модуля должен позволять позже подменить источник на БД без изменения кода бота.
- Проверка авторизации через этот модуль: только разрешённые пользователи могут отправлять команды.
- Вызов логики переключения URL (Этап 3) и возврат подтверждения или ошибки (система не готова, Chrome/OBS недоступны).
- Логирование всех входящих команд и отказ неавторизованным.

## Контекст

- Документ плана: `docs/plan-execution.md`, Этап 6.
- Зависимость: переключение URL (Этап 3), конфигурация (токен бота, список пользователей — Этап 1).
- Результат: оператор может прислать ссылку боту и получить ответ; неавторизованные отклоняются.
