# Задача: Переключение URL через CDP

## Суть задачи

Реализовать переключение страницы в уже открытом окне Chrome через Chrome DevTools Protocol: подключение к DevTools, выбор активного target, навигация по `Page.navigate`. Сохранять последний успешный URL для восстановления после рестарта. Валидация URL в MVP не требуется.

## Детали и требования

- Подключение к DevTools (порт из конфигурации), выбор активного target (нужная вкладка/окно).
- Команда навигации: `Page.navigate` по переданному URL.
- Валидация URL в MVP не требуется; при необходимости оставить место под проверку схем (http/https) и whitelist доменов в следующих итерациях.
- Сохранение последнего успешного URL для восстановления после рестарта.

## Контекст

- Документ плана: `docs/plan-execution.md`, Этап 3.
- Зависимость: модуль Chrome (Этап 2) уже запускает браузер и даёт доступ к DevTools.
- Результат: по команде (внутренней или тестовой) сервис переключает страницу в уже открытом окне Chrome.
