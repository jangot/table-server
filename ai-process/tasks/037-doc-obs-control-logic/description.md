# Задача: Документ — логика управления OBS (без запуска)

## Суть задачи
Собрать и зафиксировать в одном MD-документе полное описание логики управления OBS в текущем POC-приложении. Документ предназначен для переноса в другой репозиторий: новое приложение не будет запускать OBS и не будет контролировать его запуск, а будет только управлять уже запущенным OBS. Цель — получить подробное описание того, **чем** и **как** приложение управляет OBS (API, сцены, проектор, WebSocket и т.д.), без настроек и параметров запуска OBS.

## Детали и требования
- **Результат:** один MD-файл с подробным описанием логики управления OBS.
- **Содержание документа (что собрать из кодовой базы):**
  - Подключение к OBS (WebSocket: хост, порт, пароль; переподключение при необходимости).
  - Получение данных: список сцен, текущая программа/сцена, список мониторов и т.п.
  - Действия управления: переключение сцены (program), открытие source projector на указанный монитор, любые другие команды к OBS.
  - Роли сцен (main / output / input), как они используются в логике переключения и вывода на проектор.
  - Конфигурация, используемая именно для **управления** (имена сцен, имя монитора проектора, конфиг сцен для отображения/переключения) — без параметров запуска OBS (путь к бинарнику, аргументы и т.д.).
  - Зависимости: какие модули/сервисы вызывают OBS-логику, откуда приходят команды (HTTP, Telegram и т.д.).
- **Не включать в документ:** параметры и настройки **запуска** OBS (путь к исполняемому файлу, аргументы командной строки, переменные окружения для старта процесса). Сбор параметров запуска вынесен в отдельную задачу.
- Документ на русском или английском (по соглашению проекта переноса), структурированный, с заголовками и перечнями для удобного копирования в другой проект.

## Контекст
- Проект — POC для управления OBS и Chrome; управление OBS реализовано (сцены, проектор, WebSocket). В новом репозитории будет только управление уже запущенными OBS и Chrome, без запуска процессов.
- Релевантные области кода: модули OBS (obs-scenes, obs-websocket, конфиг OBS), вызовы из idle-server и telegram-bot.
