# Задача: OBS WebSocket и сервис управления сценами

## Суть задачи

Добавить подключение к OBS по **OBS WebSocket API** и реализовать сервис управления сценами: получение списка сцен и текущей сцены из OBS, переключение сцены по имени. Обеспечить обработку недоступности OBS и разрыва соединения, логирование в формате key=value. Источник истины по сценам — OBS (приложение сцены не создаёт).

## Детали и требования

- **Библиотека:** использовать OBS WebSocket API (например `obs-websocket-js` или аналог). Установка соединения, проверка состояния, переподключение при разрыве.
- **Конфигурация:** переменные окружения `OBS_HOST`, `OBS_PORT`, `OBS_PASSWORD` — добавить в модуль конфигурации и валидацию (обязательные при включённом управлении сценами; уточнить связь с существующим запуском OBS — один порт или отдельный).
- **API сервиса:** получение списка имён сцен из OBS; получение текущей активной сцены; переключение на сцену по имени. При запросе переключения на несуществующую сцену — понятная ошибка (не падение).
- **Ошибки:** OBS недоступен, WebSocket разорван, сцена не найдена — возвращать/логировать понятные сообщения.
- **Логирование (§13 требований):** формат плоский key=value для grep. События: подключение к OBS, восстановление соединения, переключение сцен. Пример: `scene_switch source=telegram from=chrome to=capture success=true`, `obs_connection status=reconnected`.
- **Интеграция с приложением:** сервис сцен должен быть доступен для Telegram-бота и Web API (задачи 014, 015); запуск процесса OBS остаётся в существующем модуле `obs`, не смешивать с WebSocket-логикой в одном месте.

## Контекст

- Требования: [docs/requirements/obs-scene-requirements.md](../../../docs/requirements/obs-scene-requirements.md), разделы 5–6, 10–11, 13–14.
- Текущий модуль OBS: `src/modules/obs/` — только запуск процесса OBS (launch, args, restart). WebSocket и управление сценами добавляются отдельно (новый модуль или подмодуль в `obs`, без смешивания с launch).
- Конфиг: `src/modules/config/` — добавить поля для OBS WebSocket (host, port, password). План: [docs/plan-obs-scenes.md](../../../docs/plan-obs-scenes.md).

## Открытые вопросы

- Один порт для запуска OBS и WebSocket или раздельные настройки (в требованиях пример `OBS_PORT=4455` — обычно это порт WebSocket; путь к исполняемому файлу OBS уже есть как `OBS_PATH`). При необходимости развести OBS_WEBSOCKET_HOST/PORT и OBS_HOST/OBS_PORT по документации.
