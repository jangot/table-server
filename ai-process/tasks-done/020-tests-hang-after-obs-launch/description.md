# Задача: Тесты зависают после запуска OBS

## Суть задачи
При запуске тестов они успешно стартуют и открывают OBS, однако после этого выполнение зависает и тесты не завершаются. Необходимо разобраться, почему тесты не могут продолжить выполнение после запуска OBS, и устранить причину зависания.

## Детали и требования
- Тесты запускаются штатным образом
- OBS успешно открывается в процессе выполнения тестов
- После открытия OBS тесты зависают и не продвигаются дальше
- Необходимо обеспечить корректное завершение тестов без зависания

## Контекст
Проект использует OBS для управления сценами через WebSocket. Тесты, вероятно, инициируют запуск OBS как часть проверки функциональности. Возможные причины зависания:
- Тест ожидает подключения к OBS WebSocket, но соединение не устанавливается в разумное время
- OBS не завершает инициализацию и тест не получает нужного события
- Отсутствует таймаут или mock для OBS в тестовом окружении
- Тест блокируется на ожидании готовности OBS

## Открытые вопросы
- Какие именно тесты зависают (unit, e2e, integration)?
- Используется ли реальный OBS или mock в тестах?
- Есть ли таймауты на подключение к OBS WebSocket в тестовом окружении?
- Нужно ли мокировать запуск OBS в тестах или запускать реальный?
