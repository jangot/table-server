# Задача: Фильтрация сцен по префиксу `src.` в Telegram-боте

## Суть задачи
В Telegram-боте команда `/scenes` возвращает все отображаемые сцены из OBS. Необходимо добавить фильтрацию: возвращать только сцены, имя которых начинается с префикса `src.`. При этом в ответе пользователю префикс `src.` нужно отрезать, показывая имена без него.

Команда `/scene <name>` принимает имя сцены. Необходимо изменить её так, чтобы пользователь вводил имя без префикса `src.`, а бот подставлял его автоматически перед поиском и переключением.

## Детали и требования
- `/scenes` — возвращать только сцены с именем вида `src.*`
- В ответе `/scenes` отображать имена без `src.` (например, `src.table` → `table`)
- `/scene <name>` — принимать имя без префикса (например, `/scene table`), подставлять `src.table` при поиске и переключении
- Фильтрация происходит на уровне обработчиков Telegram-бота (`handlers.ts`), а не в `ObsScenesService`
- Поле `title` из конфига сцены по-прежнему может использоваться для отображения (если задано)

## Контекст
- Обработчики команд: `src/modules/telegram-bot/handlers.ts` — функции `handleScenes` и `handleScene`
- Сервис сцен: `src/modules/obs-scenes/` — `getScenesForDisplay()` возвращает `SceneForDisplay[]` с полями `name`, `title`, `type`, `enabled`
- Текущая логика `handleScenes` (строка 153–166): получает все сцены через `getScenesForDisplay()`, форматирует список
- Текущая логика `handleScene` (строка 169–204): проверяет наличие сцены среди доступных через `getScenesForDisplay()`, затем переключает через `setScene(name)`
- Сцены в OBS называются по шаблону `src.<shortname>` (например, `src.table`, `src.camera`)
