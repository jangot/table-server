# Задача: Watchdog и восстановление Chrome/OBS

## Суть задачи

Реализовать периодическую проверку состояния Chrome и OBS через соответствующие модули и при падении выполнять рестарт (Chrome — через модуль Chrome, OBS — через модуль OBS) с ограничением частоты рестартов. При падении Chrome — открытие последнего URL или idle-страницы после восстановления.

## Детали и требования

- Периодическая проверка состояния через **соответствующие модули** (Chrome — модуль Chrome, OBS — модуль OBS), без смешивания логики.
- **При падении Chrome:** рестарт через модуль Chrome, ожидание готовности, при необходимости открытие последнего URL или idle-страницы (локальный EJS-сервер).
- **При падении OBS:** рестарт через модуль OBS (как в Этапе 4).
- Ограничение частоты рестартов (backoff), статусы в логах (например, `ready` / `degraded` при частичной готовности).

## Контекст

- Документ плана: `docs/plan-execution.md`, Этап 5.
- Зависимость: модули Chrome (Этап 2), переключение URL (Этап 3), модуль OBS (Этап 4). Idle-страница уже поднята на Этапе 1.
- Результат: система восстанавливается после падения Chrome или OBS без бесконечных циклов рестарта; проверки и рестарты идут через отдельные модули приложений.
