# Задача: Запуск и контроль Chrome

## Суть задачи

Реализовать отдельный модуль для работы с Chrome: запуск процесса с remote debugging, ожидание готовности DevTools, поддержка одного окна/сессии. Модуль должен выдавать сигнал «ready» для оркестратора и не содержать логики OBS или других приложений.

## Детали и требования

- **Отдельный модуль** только для Chrome (логика OBS и других приложений в этот модуль не входит).
- **Запуск Chrome:** аргументы из конфигурации (remote debugging port, режим окна — kiosk/app/fullscreen); без передачи пользовательского ввода в shell (защита от command injection).
- **Ожидание готовности:** проверка процесса и доступности DevTools endpoint (запрос к `/json/version` или подключение к порту). Модуль предоставляет способ «дождаться старта» (Promise/колбэк «ready»), чтобы оркестратор запускал следующее приложение только после этого.
- **Одно окно/сессия:** не создавать лишние окна и вкладки.
- При необходимости: выбор нужного target по CDP и сохранение последнего открытого URL в состоянии.

## Контекст

- Документ плана: `docs/plan-execution.md`, Этап 2.
- Зависимость: после Этапа 1 (оркестрация и конфигурация). Пути и порты берутся из конфигурации (env).
- Результат: модуль Chrome запускает процесс, дожидается готовности DevTools, экспортирует состояние «готов» для оркестрации.
