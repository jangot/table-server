# Задача: Документ — логика управления Chrome (без запуска)

## Суть задачи
Собрать и зафиксировать в одном MD-документе полное описание логики управления Chrome в текущем POC-приложении. Документ предназначен для переноса в другой репозиторий: новое приложение не будет запускать Chrome и не будет контролировать его запуск, а будет только управлять уже запущенным Chrome. Цель — получить подробное описание того, **чем** и **как** приложение управляет Chrome (CDP, вкладки, URL, окна, скрипты на странице и т.д.), без настроек и параметров запуска Chrome.

## Детали и требования
- **Результат:** один MD-файл с подробным описанием логики управления Chrome.
- **Содержание документа (что собрать из кодовой базы):**
  - Подключение к Chrome (CDP/DevTools: endpoint, порт, переподключение при необходимости).
  - Получение данных: список вкладок/окон, текущий URL, размеры окна и т.п.
  - Действия управления: переключение URL, выполнение скриптов в контексте страницы (по домену или иначе), управление вкладками/окнами, масштаб/zoom, fullscreen — всё, что реально используется в коде.
  - Привязка окна Chrome к источнику OBS (если есть в этом проекте) — только аспект управления (как определяется окно, что делается с ним), без параметров запуска Chrome.
  - Конфигурация, используемая именно для **управления** (URL по умолчанию, домены для скриптов, размер окна для привязки и т.д.) — без параметров запуска Chrome (путь к бинарнику, флаги, kiosk и т.д.).
  - Зависимости: какие модули/сервисы вызывают Chrome-логику, откуда приходят команды (HTTP, Telegram и т.д.).
- **Не включать в документ:** параметры и настройки **запуска** Chrome (путь к исполняемому файлу, флаги командной строки, kiosk, autoplay и т.п.). Сбор параметров запуска вынесен в отдельную задачу.
- Документ на русском или английском (по соглашению проекта переноса), структурированный, с заголовками и перечнями для удобного копирования в другой проект.

## Контекст
- Проект — POC для управления OBS и Chrome; управление Chrome реализовано (CDP, смена URL, скрипты по доменам и т.д.). В новом репозитории будет только управление уже запущенным Chrome, без запуска процесса.
