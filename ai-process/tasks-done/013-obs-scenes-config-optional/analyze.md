# Анализ: Опциональный конфиг сцен для UI

## Общее описание функциональности

Нужно добавить **опциональную** поддержку файла конфигурации сцен (`SCENES_CONFIG_PATH`): JSON с полями `name`, `title`, `type`, `enabled` для обогащения списка сцен при отображении в UI (подписи, типы, возможность скрывать сцены в UI). Источник истины по наличию и переключению сцен остаётся OBS; при отсутствии конфига или несовпадении имён — не ошибка, приложение работает только по данным из OBS.

Цель: дать UI (Telegram, GET /obs/scenes в задачах 014 и 016) возможность показывать человекочитаемые подписи и типы сцен, а также фильтровать по `enabled`, не меняя логику переключения (она по-прежнему опирается на список сцен из OBS).

## Текущие интерфейсы и API (если есть)

- **ObsScenesService** (`src/modules/obs-scenes/types.ts`): `getScenes(): Promise<string[]>`, `getCurrentScene(): Promise<string | null>`, `setScene(name: string): Promise<void>`. Сейчас `getScenes()` возвращает только массив имён из OBS.
- **createObsScenesService** (`src/modules/obs-scenes/index.ts`): принимает `ObsConfig` и `Logger`, создаёт клиент WebSocket и реализацию сервиса; конфиг сцен не передаётся.
- **createObsScenesServiceImpl** (`src/modules/obs-scenes/scenes-service.ts`): принимает `ObsScenesServiceConfig` с `client` и `logger`; получает список сцен только из OBS.
- **ObsConfig** (`src/modules/config/types.ts`): поля для OBS (path, readyTimeout, profilePath, host, port, password); поля для конфига сцен нет.
- **validateEnv** (`src/modules/config/validate.ts`): собирает конфиг из env; опциональные строковые пути обрабатываются как `getEnv('X')?.trim() || undefined` (например `lastUrlStatePath`).

Потребители списка сцен (после 013): команды Telegram (014) и страница/API GET /obs/scenes (015, 016). Им понадобится обогащённый список (name + title, type, enabled), а не только имена.

Уточнение по месту хранения пути: в требованиях §14 переменная окружения `SCENES_CONFIG_PATH` опциональна. Достаточно читать её в validate (как lastUrlStatePath) и передавать в фабрику сервиса сцен; отдельного класса валидации для этого пути не требуется.

## Зависимости и ограничения

- **Задача 012** уже реализована: сервис сцен возвращает список имён из OBS и переключает сцену. Задача 013 не меняет источник истины — только обогащает данные для отображения.
- **Формат конфига** (§5 требований): массив объектов `{ "name", "title", "type", "enabled" }`. Имена в конфиге могут не совпадать со сценами в OBS — такие записи не используются для переключения; список сцен для переключения и отображения по наличию в OBS.
- **Поведение при ошибках**: отсутствие файла, пустой конфиг, невалидный JSON, несовпадение имён конфига со сценами OBS — приложение не падает, полностью полагается на OBS для списка и переключения.
- **enabled**: использовать для фильтрации в UI (скрывать сцены с enabled: false), не блокируя переключение по API (т.е. переключение по имени остаётся возможным и без конфига).
- **Документация**: docs/requirements/obs-scene-requirements.md (§5, §14), docs/plan-obs-scenes.md (этап 2).

Риски: чтение файла при каждом запросе списка сцен vs при старте — при необходимости кэшировать содержимое конфига или перечитывать по таймеру/при первом запросе, чтобы не усложнять первую версию.
