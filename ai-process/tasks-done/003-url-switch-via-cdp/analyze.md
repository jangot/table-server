# Анализ: Переключение URL через CDP

## Общее описание функциональности

Нужно реализовать переключение страницы в уже открытом окне Chrome через Chrome DevTools Protocol: подключение к DevTools по порту из конфигурации, выбор активного target (вкладка/окно), навигация командой `Page.navigate` по переданному URL. В MVP валидация URL не требуется. Последний успешный URL должен сохраняться для восстановления после рестарта сервиса (используется на Этапе 5 — watchdog при рестарте Chrome и на Этапе 7 — восстановление при рестарте сервиса).

Итог: по команде (внутренней или тестовой) сервис переключает страницу в уже открытом окне Chrome.

## Связанные модули и сущности

| Модуль/файл | Назначение | Что затрагивает задача |
|-------------|------------|------------------------|
| `src/modules/chrome/` | Запуск Chrome, ожидание DevTools, хранение процесса | Основная зона изменений: CDP-подключение, выбор target, `Page.navigate`, сохранение последнего URL (в памяти и на диск) |

| `src/modules/logger/` | Логирование | Использование при навигации и ошибках CDP |

## Текущие интерфейсы и API (если есть)

- **`AppModule`** (`orchestrator`): `{ name, start() }` — модуль Chrome уже реализует; расширять интерфейс оркестратора под задачу 003 не требуется.
- **`createChromeModule(config, logger)`** — возвращает модуль с `start()`, который запускает Chrome и ждёт готовности DevTools. После старта нет API для навигации; его нужно добавить (например, функция `navigateToUrl(url: string): Promise<void>` или объект с методом, доступный после готовности Chrome).
- **`getChromeProcess()`** (`chrome/launch.ts`) — экспортируется «for use in task 003 (CDP / close)»; можно использовать для проверки, что Chrome запущен, перед вызовом CDP.
- **`AppConfig`** (`config/types.ts`): уже есть `devToolsPort?: number` (порт DevTools, по умолчанию 9222 в модуле Chrome). Этого достаточно для подключения к CDP; при необходимости в конфиг можно добавить путь к файлу для сохранения последнего URL.
- **Готовность DevTools**: проверяется через HTTP `GET http://127.0.0.1:${port}/json/version` в `waitForDevTools`. Для CDP нужен WebSocket: список target’ов получают через `GET http://127.0.0.1:${port}/json` или `/json/list`, в ответе — `webSocketDebuggerUrl` для каждого target; к одному из них подключаются по WebSocket и вызывают `Page.navigate`.

В проекте **нет** зависимостей для работы с CDP (ни puppeteer, ни chrome-remote-interface); для вызова `Page.navigate` потребуется либо добавить библиотеку (например `puppeteer-core` только для `connect()`, либо `chrome-remote-interface`), либо реализовать минимальный WebSocket-клиент и отправку JSON-RPC вручную.

## Зависимости и ограничения

- **Зависимость от Этапа 2:** модуль Chrome уже запускает браузер с `--remote-debugging-port` и дожидается готовности DevTools по HTTP. Задача 003 опирается на этот порт (`config.devToolsPort ?? 9222`) и на то, что к моменту вызова навигации Chrome уже запущен.
- **Отсутствие CDP-клиента:** в `package.json` нет библиотек для CDP. Нужно добавить зависимость: например `puppeteer-core` (только для `connect({ port })` и вызова CDP) или `chrome-remote-interface`. Альтернатива — минимальная реализация через WebSocket и JSON-RPC вручную (больше кода, меньше зависимостей).
- **Выбор target:** в сценарии одно окно/одна вкладка; достаточно выбрать один page-type target (например, первый из списка с `type: 'page'`). Получение списка — через HTTP `GET http://127.0.0.1:${port}/json` или `/json/list`.
- **Валидация URL:** в MVP не требуется; оставить возможность расширения (например, отдельная функция-заглушка или комментарий про схемы/whitelist в следующих итерациях).
- **Сохранение последнего URL:** нужно определить место хранения (файл). При рестарте сервиса или Chrome этот URL будет использоваться для открытия страницы (логика восстановления — этапы 5 и 7); в рамках 003 достаточно реализовать запись после успешной навигации и чтение при старте/по запросу.
- **Риски:** при падении Chrome во время навигации CDP-клиент может зависнуть или выдать ошибку — обрабатывать таймауты и разрыв соединения; не блокировать основной цикл сервиса.
