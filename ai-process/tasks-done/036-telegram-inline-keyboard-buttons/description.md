# Задача: Telegram inline-кнопки вместо команд

## Суть задачи

Преобразовать Telegram-команды бота в inline-кнопки (InlineKeyboard). В первую очередь — команда `/scenes`: вместо текстового bullet-списка сцен должен возвращаться список inline-кнопок, каждая из которых переключает OBS на соответствующую сцену. Остальные команды без параметров также можно оформить в виде кнопок через команду `/menu` или аналогичную точку входа.

## Детали и требования

- **Команда `/scenes`:**
  - Вместо текстового списка сцен возвращает сообщение с `inline_keyboard`
  - Каждая кнопка соответствует одной сцене (фильтр по префиксу `src.` сохраняется)
  - Текст кнопки: `s.title ?? s.name.slice(4)` (как сейчас в списке)
  - `callback_data` кнопки должен содержать имя сцены для переключения (например, `scene:src.CameraFeed`)
  - При нажатии кнопки — выполняется логика переключения сцены (аналог `handleScene`)

- **Обработка нажатий (`callback_query`):**
  - Добавить `bot.action(...)` в `run.ts` для обработки нажатий на кнопки сцен
  - Проверять авторизацию пользователя так же, как в командах
  - Отвечать подтверждением переключения или ошибкой

- **Остальные команды (опционально, по договорённости):**
  - Команды без параметров (`/status`, `/idle`, `/current`, `/backup`, `/default`, `/help`) могут быть представлены как кнопки в отдельном `/menu`
  - Команда `/restart` требует параметр (`chrome` | `obs` | `all`) — её кнопки тоже можно сделать inline

## Контекст

- Бот реализован через **Telegraf**
- Все команды зарегистрированы в `src/modules/telegram-bot/run.ts`
- Хендлеры — в `src/modules/telegram-bot/handlers.ts`
- Ни кнопок, ни InlineKeyboard в коде сейчас нет — только `ctx.reply(text)`
- Структура сцены: `{ name: string, title?: string, type?: string, enabled?: boolean }`, возвращается через `deps.obsScenes.getScenesForDisplay()`
- Сцены с префиксом `src.` — основные отображаемые сцены (фильтр уже есть в `handleScenes`)
- Авторизация через `deps.allowedUsers.isAllowed({ id, username })` — обязательна для всех действий
