# Задача: Проанализировать проект и сделать README.md

## Суть задачи

Проанализировать структуру и назначение проекта table-server и подготовить полноценный README.md в корне репозитория. README должен давать новому разработчику или оператору понятное описание: что это за сервис, зачем он нужен, как его установить, настроить и запустить.

## Детали и требования

- Исходная формулировка: «проанализировать проект и сделать README.md».
- Нужно опираться на реальную структуру проекта (модули в `src/modules`, скрипты в `package.json`, документацию в `docs/`).
- README должен отражать фактическое назначение сервиса: управление OBS и Chrome для трансляции на проектор, приём команд (в т.ч. через Telegram), автозапуск и восстановление.
- Учесть наличие конфигурации (в т.ч. через .env), требований к окружению (Node.js, OBS, Chrome) и существующие команды: `npm run build`, `npm start`, `npm run dev`, `npm test`.
- Язык контента README — по правилам проекта: landing page и подобный контент на английском; для внутренней документации допустим русский — в описании задачи не уточнено, поэтому можно зафиксировать при анализе (например, README на английском для открытого репозитория или на русском для внутреннего использования).

## Контекст

- Проект: Node.js-приложение `table-server`, описание в `package.json` пустое.
- Документация требований: `docs/requirements/init.md` — полное описание целей, архитектуры и функциональных требований сервиса (OBS, Chrome, Telegram, автозапуск, watchdog).
- Модули в `src/modules/`: config, chrome, obs, telegram-bot, watchdog и др. — их назначение нужно отразить в README.
- Есть тесты, линтер, скрипты для задач в `ai-process/`.

## Открытые вопросы

- Язык README: английский или русский? (при анализе можно предложить вариант и обоснование)
- Нужны ли в README разделы про конфигурацию (переменные окружения, пример .env), про развёртывание (PM2/autostart) и про команды Telegram-бота — уточнить при анализе на основе существующей документации и кода.
