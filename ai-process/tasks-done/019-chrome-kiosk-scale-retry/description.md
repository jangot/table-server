# Задача: Повторно исправить запуск Chrome — kiosk и масштаб контента

## Суть задачи
Задача 018 (`chrome-kiosk-and-content-scale-fix`) была реализована: добавлены флаги `--noerrdialogs`, `--disable-infobars`, `--force-device-scale-factor`. Однако проблема не устранена — контент по-прежнему отображается неправильно и/или kiosk-режим не работает как ожидается. Необходимо провести повторное исследование и найти настоящую причину.

## Детали и требования
- Предыдущее решение (задача 018) реализовано полностью и уже находится в `src/modules/chrome/args.ts`
- Флаги `--kiosk`, `--noerrdialogs`, `--disable-infobars` добавлены и работают по коду
- Флаг `--force-device-scale-factor` добавлен, но требует установки через `CHROME_DEVICE_SCALE_FACTOR` в `.env`
- Проблема сохраняется несмотря на изменения — нужно найти что именно не так
- Возможные направления: неверный `.env` (переменная не установлена), Chrome запускается не через `buildChromeArgs`, дисплейный сервер не поддерживает kiosk, профиль пользователя Chrome переопределяет флаги

## Контекст
- Задача 018 (выполнена): `ai-process/tasks-done/018-chrome-kiosk-and-content-scale-fix/`
- Код аргументов Chrome: `src/modules/chrome/args.ts`
- Конфиг Chrome: `src/modules/config/types.ts`, `src/modules/config/validate.ts`
- Переменные окружения: `.env` / `.env.example`
- Среда запуска: Linux, предположительно X11 или Wayland

## Открытые вопросы
- Установлена ли переменная `CHROME_DEVICE_SCALE_FACTOR` в реальном `.env`?
- Реально ли передаются новые флаги при запуске Chrome (можно проверить через `ps aux | grep chrome`)?
- Какой именно визуальный симптом остался: маленький контент, видимые рамки, или что-то ещё?
- Используется ли `CHROME_WINDOW_MODE=kiosk` или `CHROME_KIOSK=true` в `.env`?
- Не переопределяет ли профиль пользователя Chrome (`userDataDir`) установленный масштаб?
