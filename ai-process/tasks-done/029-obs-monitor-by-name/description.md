# Задача: Идентификация монитора OBS по имени входа

## Суть задачи
Сейчас монитор для OBS-проектора задаётся через переменную окружения `OBS_PROJECTOR_MONITOR` как числовой индекс (0, 1, 2...). Это ненадёжно: порядок мониторов в системе может меняться при переподключении, перезагрузке или изменении конфигурации дисплеев. Нужно заменить идентификацию монитора по индексу на идентификацию по имени входа (например, `HDMI-1`, `DP-2`). OBS WebSocket API предоставляет метод `GetMonitorList`, который возвращает список мониторов с именами — по нему можно найти нужный монитор и получить его индекс динамически.

## Детали и требования
- Новая переменная окружения (например, `OBS_PROJECTOR_MONITOR_NAME`) — строка с именем входа монитора
- При старте / переподключении к OBS: вызвать `GetMonitorList`, найти монитор по имени, получить его индекс
- Если монитор с таким именем не найден — залогировать ошибку и не открывать проектор
- Старую переменную `OBS_PROJECTOR_MONITOR` (числовой индекс) желательно оставить как fallback или убрать — уточнить
- Валидация конфига: новое поле должно быть опциональной строкой

## Контекст
- Конфиг OBS: `src/modules/config/types.ts` (поле `projectorMonitorIndex`)
- Парсинг env: `src/modules/config/validate.ts` (строка с `OBS_PROJECTOR_MONITOR`)
- OBS WebSocket клиент: `src/modules/obs-scenes/client.ts` (метод `openSourceProjector`)
- Сервис сцен: `src/modules/obs-scenes/index.ts` (логика авто-открытия проектора при подключении)
- OBS WebSocket API метод для списка мониторов: `GetMonitorList`

## Открытые вопросы
- Оставить ли `OBS_PROJECTOR_MONITOR` (индекс) как fallback, или полностью заменить на имя?
- Какой формат имени монитора используется в целевой системе (например, `HDMI-1`, `DP-0`, или отображаемое имя из OBS)?
