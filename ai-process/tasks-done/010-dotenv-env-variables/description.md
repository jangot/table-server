# Задача: Использование dotenv для переменных окружения

## Суть задачи
Сейчас в приложении переменные окружения приходится задавать вручную через `export` в shell. Нужно перейти на использование библиотеки **dotenv**: загружать переменные из файла `.env` при старте приложения, чтобы не выполнять экспорт вручную.

## Детали и требования
- Подключить и настроить dotenv для загрузки переменных из `.env`
- Обеспечить загрузку до инициализации приложения (до чтения `process.env` в коде)
- Учесть, что `.env` не должен попадать в репозиторий (добавить в `.gitignore`, если ещё не добавлен)
- Документировать формат `.env` и пример (например `.env.example`) для разработчиков

## Контекст
- Приложение — table-server; переменные окружения используются для конфигурации (возможно, порты, ключи, пути и т.п.)
- После внедрения dotenv текущий способ с ручным `export` остаётся опциональным (переменные из окружения имеют приоритет над файлом — стандартное поведение dotenv)

## Открытые вопросы
- Нужен ли разный набор переменных для разных окружений (dev/stage/prod) или достаточно одного `.env`/`.env.example` — при необходимости уточнить по кодовой базе или с пользователем.
