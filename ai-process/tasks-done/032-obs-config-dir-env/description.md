# Задача: OBS config-dir через переменную окружения

## Суть задачи
Добавить поддержку аргумента `--config-dir` при запуске OBS, чтобы папку конфигурации OBS можно было вынести в произвольное место и указать через переменную окружения. Переменная окружения должна быть обязательной и зафиксирована в `.env.example`.

## Детали и требования
- Добавить переменную окружения `OBS_CONFIG_DIR` (название по аналогии с существующими)
- Переменная **обязательная** (не optional) — OBS не запустится без неё
- Значение передаётся как `--config-dir <path>` в аргументы при запуске OBS
- Добавить поле `configDir` в класс `ObsConfig` (`config/types.ts`) без декоратора `@IsOptional()`
- Читать `OBS_CONFIG_DIR` в `config/validate.ts` аналогично другим строковым переменным
- Добавить генерацию аргумента `--config-dir` в `obs/args.ts`
- Добавить `OBS_CONFIG_DIR=` в `.env.example` с комментарием

## Контекст
- Существующий аналог: `OBS_PROFILE_PATH` — аналогичная переменная, но опциональная, передаётся как `--profile=<path>`
- Файлы изменений: `src/modules/config/types.ts`, `src/modules/config/validate.ts`, `src/modules/obs/args.ts`, `.env.example`
- OBS поддерживает `--config-dir <path>` как стандартный CLI-аргумент
