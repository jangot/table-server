# Задача: SSR-страница управления сценами OBS

## Суть задачи

Реализовать страницу **GET /obs/scenes** с отображением статуса подключения к OBS, текущей сцены, списка доступных сцен и кнопок переключения (включая отдельные кнопки Backup и Default). Активная сцена должна быть визуально выделена. Рендеринг на сервере (EJS), как существующая idle-страница.

## Детали и требования

- **Маршрут:** GET /obs/scenes (§8 требований).
- **Содержимое страницы:**
  - Статус подключения к OBS (connected/disconnected или аналог).
  - Текущая сцена (название/подпись).
  - Список доступных сцен — кнопки для переключения на каждую сцену.
  - Дополнительные кнопки: **Backup**, **Default**.
  - Активная сцена визуально выделена (стиль/состояние кнопки).
- **Реализация:** Express + EJS. Данные для шаблона: статус OBS, текущая сцена, список сцен (из сервиса сцен с учётом опционального конфига task 013). Переключение при нажатии кнопок — через POST-запросы к Web API (POST /obs/scene и т.д.) или прямой вызов сервиса сцен с последующим редиректом/обновлением страницы.
- **Ошибки:** при недоступности OBS показывать это на странице (например «OBS disconnected»), не падать.

## Контекст

- Требования: [docs/requirements/obs-scene-requirements.md](../../../docs/requirements/obs-scene-requirements.md), §8.

- Web API переключения — задача 015 (POST /obs/scene, /obs/scene/backup, /obs/scene/default). Страница может вызывать их через формы или fetch и обновлять/редирект на GET /obs/scenes.
