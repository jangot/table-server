# Задача: Запуск и контроль OBS

## Суть задачи

Реализовать отдельный модуль для работы с OBS: запуск только после готовности Chrome (через оркестратор), запуск с сохранённой конфигурацией/сценой, детекция падения и перезапуск с ограничением частоты. Логика Chrome и других приложений в этот модуль не входит.

## Детали и требования

- **Отдельный модуль** только для OBS (логика Chrome и других приложений в этот модуль не входит).
- **Порядок запуска:** OBS запускается только после готовности Chrome; оркестратор дожидается «ready» от модуля Chrome, затем вызывает запуск OBS из модуля OBS.
- **Запуск OBS:** с сохранённой конфигурацией/сценой (путь из конфигурации/env).
- **Детекция готовности:** определение, запущен ли OBS (процесс и/или порт, если применимо); модуль при необходимости предоставляет сигнал готовности для последующих звеньев цепочки.
- **При падении OBS:** перезапуск с ограничением частоты рестартов и логированием причины (внутри модуля OBS).

## Контекст

- Документ плана: `docs/plan-execution.md`, Этап 4.
- Зависимость: оркестрация (Этап 1) и модуль Chrome (Этап 2). Порядок «сначала Chrome, потом OBS» обеспечивается оркестрацией, а не смешиванием кода в одном модуле.
- Результат: модуль OBS запускает OBS, детектирует падение и перезапускает; порядок запуска обеспечивается оркестратором.
