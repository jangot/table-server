# Задача: Контент в Chrome отображается маленьким в полноэкранном режиме

## Суть задачи
Chrome открывается корректно — в полноэкранном режиме без элементов интерфейса. Однако контент внутри Chrome отображается не на всю доступную область, а как небольшой прямоугольник на большом белом (пустом) холсте. Нужно разобраться, почему viewport или масштаб страницы не соответствует реальному размеру окна Chrome, и устранить проблему.

## Детали и требования
- Chrome запускается в kiosk-режиме (или fullscreen), без UI-элементов — это работает правильно
- Содержимое страницы выглядит маленьким относительно доступного пространства — как будто viewport или zoom не соответствует размеру экрана
- Визуально: небольшой прямоугольник с контентом на большом пустом холсте
- Нужно добиться, чтобы контент занимал всю доступную область в Chrome

## Контекст
- Ранее уже были задачи 018 и 019, связанные с масштабом контента в kiosk-режиме (`chrome-kiosk-and-content-scale-fix`, `chrome-kiosk-scale-retry`) — стоит изучить, что там делалось и почему проблема не решена до конца
- Возможные причины: неверный `--force-device-scale-factor`, проблемы с CSS viewport meta, несоответствие разрешения экрана и размера окна Chrome, или параметры `--window-size`
