# Задача: Команды в Telegram не возвращают сцены OBS

## Суть задачи
Команды в Telegram-боте сообщают, что сцен нет, хотя сцены в OBS существуют и настроены. Нужно выяснить причину расхождения и обеспечить корректное получение и отображение списка сцен при запросе через Telegram.

## Детали и требования
- Команды в Telegram не возвращают сцены (пользователь видит сообщение об отсутствии сцен).
- В OBS сцены присутствуют и доступны.
- Требуется синхронизация или корректное получение списка сцен при обращении через бота.

## Контекст
- Связано с Telegram-ботом и интеграцией OBS (вероятно OBS WebSocket API и/или конфигурация сцен).
- В проекте уже есть задачи по сценам: 014-telegram-scene-commands, 015-obs-scenes-web-api, 012-obs-websocket-and-scene-service, 013-obs-scenes-config-optional, 016-obs-scenes-ssr-page.

## Открытые вопросы
- В какой момент теряется список сцен: при старте приложения, при каждом запросе или после перезапуска OBS?
- Используется ли кэш сцен и может ли он быть устаревшим?

## Решение

Причина: в `.env` отсутствовали переменные `OBS_HOST`, `OBS_PORT`, `OBS_PASSWORD`.

Функция `createObsScenesService` проверяет `isObsScenesEnabled(config)`, которая требует чтобы все три параметра были заданы. Без них функция возвращает `null`, который превращается в `undefined` в `deps.obsScenes` — и бот отвечал "OBS scenes недоступны".

Исправление: добавлены переменные в `.env`:
- `OBS_HOST=localhost`
- `OBS_PORT=4455`
- `OBS_PASSWORD=<пароль из OBS WebSocket Server Settings>`

Также добавлены в `.env.example` для документирования.
