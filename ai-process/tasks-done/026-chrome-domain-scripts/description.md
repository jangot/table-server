# Задача: Скрипты по доменам/URL при открытии ссылок в Chrome

## Суть задачи
На некоторых сайтах для попадания на нужный экран необходимо выполнить дополнительные действия (логин, клики, принятие соглашений и т.п.). Нужно добавить возможность привязывать скрипты к доменам или конкретным URL, которые выполняются в Chrome при открытии соответствующей страницы.

## Детали и требования
- Реестр скриптов — папка с JS-файлами (один файл = один скрипт).
- Мап домен → имя файла скрипта: указывает, какой скрипт запустить для домена.
- При навигации Chrome на URL — извлекается домен, ищется в мапе. Если есть совпадение — читается соответствующий JS-файл из папки реестра и выполняется в контексте страницы через CDP. Если домена в мапе нет — ничего не делаем.
- Скрипты на JavaScript, выполняются в контексте страницы.
- Цель — автоматизировать действия (логин, клики, принятие соглашений) для попадания на нужный экран.

## Контекст
- Управление Chrome в проекте: 002-chrome-launch-and-control, 003-url-switch-via-cdp. CDP (Chrome DevTools Protocol) подходит для внедрения скриптов и выполнения действий на странице.

## Архитектура
- Папка реестра скриптов: путь задаётся через env-переменную (например `CHROME_SCRIPTS_DIR`).
- Мап домен → файл: JSON-файл рядом или фиксированное имя в папке реестра (например `domains.json`), путь через env-переменную (например `CHROME_SCRIPTS_MAP`).
- При старте/навигации: загружается мап, по домену URL ищется файл, скрипт читается и выполняется через `page.evaluate()`.
