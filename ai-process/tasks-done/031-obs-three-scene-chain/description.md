# Задача: Реализовать цепочку из трёх уровней сцен OBS

## Суть задачи
Сейчас команды Telegram и веб переключают программную сцену OBS (`SetCurrentProgramScene`), тогда как на проектор выводится сцена `main`, открытая через source projector. Переключение поэтому не влияет на то, что видно на проекторе. Нужно реализовать цепочку из трёх уровней: сцены с контентом → сцена output.* → сцена main на проекторе. Команды должны переключать, какую сцену с контентом показывается внутри сцены output.* (через вложенный источник типа «Сцена»), а не программную сцену.

## Детали и требования
- **Цепочка сцен:**
  1. **Сцены с контентом** (table, cam, slides и т.д.) — то, что переключается по командам.
  2. **output.*** — одна сцена-«оболочка», внутри которой один источник типа «Сцена» (nested scene) показывает одну из сцен с контентом.
  3. **main** — сцена на проекторе; в ней источник типа «Сцена» показывает сцену output.*.
- **Требование к переключению:** Команды Telegram (`/scene`, `/backup`, `/default`) и веб-интерфейс `/obs/scenes` должны переключать **вложенную сцену внутри output.***, а не вызывать `SetCurrentProgramScene`. То есть через OBS WebSocket нужно менять, какую сцену с контентом показывает источник-сцена внутри сцены output.* (например, через `SetInputSettings` у соответствующего input).
- **Конфигурация:** Нужно задать имя сцены output.* (одна сцена уровня output), в которой находится вложенный источник для переключения — через переменную окружения и/или конфиг сцен.
- **Обратная совместимость:** Поведение при отсутствии настройки цепочки (например, нет output-сцены или вложенного источника) должно быть предсказуемым: логирование, возможно fallback на текущее поведение `SetCurrentProgramScene` или отказ переключения.

## Контекст
- В задаче 030 реализовано: на проектор выводится сцена `main` (или из `OBS_PROJECTOR_SCENE_NAME`), список для переключения формируется из сцен с типами output/input, команды вызывают `obsScenes.setScene(name)` → `SetCurrentProgramScene(name)`. Проектор при этом показывает только main и не реагирует на смену программной сцены.
- OBS WebSocket (obs-websocket-js, v5): для переключения вложенной сцены внутри другой сцены используется работа с inputs (источники сцены имеют настройку с именем показываемой сцены), например `GetInputSettings` / `SetInputSettings` или аналог в актуальном протоколе.
- Модули: `src/modules/obs-scenes/` (client, scenes-service, index), `src/modules/telegram-bot/handlers.ts`, `src/modules/idle-server/index.ts`, конфиг (ObsConfig, SCENES_CONFIG_PATH, OBS_PROJECTOR_SCENE_NAME).

## Открытые вопросы
- Точное имя метода/параметра в OBS WebSocket v5 для установки сцены у источника типа «Сцена» (GetInputSettings/SetInputSettings и имя поля).
- Как однозначно идентифицировать в сцене output.* единственный источник типа «Сцена» (по имени источника, по индексу, по типу) — зависит от соглашений в OBS у пользователя.
- Нужен ли fallback на `SetCurrentProgramScene`, если вложенное переключение недоступно (например, старая схема без output/main цепочки).
