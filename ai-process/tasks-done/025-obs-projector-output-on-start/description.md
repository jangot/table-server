# Задача: Трансляция OBS на проектор (второй экран) при запуске

## Суть задачи
После перезапуска OBS трансляция на второй монитор (Fullscreen Projector) не восстанавливается — её нужно настраивать вручную. Нужно сделать так, чтобы при старте приложения (и при каждом переподключении к OBS) система автоматически открывала Fullscreen Projector нужной сцены на нужном мониторе. Номер монитора задаётся через переменную окружения.

## Детали и требования

- Соглашение об именовании сцен: `output.*` — сцены вывода, `src.*` — источники.
- Сцена `output.*` агрегирует в себе все `src.*` сцены как источники.
- При старте система находит **первую сцену с префиксом `output.`** и считает её основной выходной сценой.
- Для неё вызывается OBS WebSocket метод `OpenSourceProjector` (obs-websocket v5), что открывает Fullscreen Projector на заданном мониторе.
- Номер монитора (`monitorIndex`) задаётся через **переменную окружения** (например, `OBS_PROJECTOR_MONITOR`).
- Логика должна срабатывать как при первом подключении к OBS WebSocket, так и при каждом **переподключении** (reconnect).
- Пока поддерживается только одна output-сцена и один монитор для проектора.

## Контекст

- В проекте есть OBS WebSocket клиент: `src/modules/obs-scenes/client.ts`
  - Уже есть метод `call(method, data)` для произвольных вызовов OBS API
  - Есть хук успешного подключения в `tryConnect()` — туда подключается логика reconnect
- Сервис сцен: `src/modules/obs-scenes/scenes-service.ts`
- Конфиг OBS: `src/modules/config/types.ts` → класс `ObsConfig`

### Предлагаемый план реализации
1. Добавить `projectorMonitorIndex?: number` в `ObsConfig` (читается из env-переменной через существующий механизм конфига).
2. Добавить `onConnected?: () => Promise<void>` callback в `ObsWebSocketClientConfig` — вызывается при connect и reconnect.
3. Добавить метод `openSourceProjector(sourceName: string, monitorIndex: number): Promise<void>` в `ObsWebSocketClient`.
4. В `createObsScenesServiceImpl`: если `projectorMonitorIndex` задан — при `onConnected` получить список сцен, найти первую `output.*`, вызвать `openSourceProjector`.

## Открытые вопросы

- Как именно называется переменная окружения для номера монитора — уточнить при реализации (например, `OBS_PROJECTOR_MONITOR`).
- Что делать, если `output.*` сцен не найдено — логировать предупреждение и пропускать.
- Нужно ли открывать проектор при каждом reconnect или только при первом подключении? (Предположительно — при каждом, т.к. OBS перезапускается и состояние сбрасывается.)
