# Задача: Размер и позиция окна Chrome из переменных окружения

## Суть задачи
При запуске Google Chrome нужно иметь возможность задавать размер окна (ширину и высоту) и позицию окна (X, Y) через переменные окружения. Если переменные заданы — при старте Chrome должен получать аргументы `--window-size=WIDTH,HEIGHT` и при необходимости `--window-position=X,Y`. Если не заданы — поведение без изменений (Chrome использует размер и позицию по умолчанию).

## Детали и требования
- Добавить поддержку размера и позиции окна в конфигурации Chrome (секция `chrome` в `AppConfig`).
- **Размер:** задаётся переменными окружения (например `CHROME_WINDOW_WIDTH` и `CHROME_WINDOW_HEIGHT`, или одна переменная в формате `WIDTHxHEIGHT` — на этапе анализа/плана можно уточнить формат). В `buildChromeArgs` при наличии заданного размера добавлять аргумент `--window-size=WIDTH,HEIGHT`.
- **Позиция:** задаётся переменными окружения (например `CHROME_WINDOW_POSITION_X` и `CHROME_WINDOW_POSITION_Y`, или одна переменная в формате `X,Y` — формат уточнить в плане). В `buildChromeArgs` при наличии заданной позиции добавлять аргумент `--window-position=X,Y`.
- Валидация размера: оба значения — целые положительные числа в разумных пределах (например 1–7680 для типичных разрешений). Если задана только одна из сторон — либо не применять размер, либо явно описать поведение в требованиях/плане.
- Валидация позиции: X и Y — целые числа (допустимы отрицательные для мультимониторных конфигураций), в разумных пределах. Если задана только одна координата — не применять позицию или описать поведение в плане.
- Документация: обновить `docs/architecture/config.md` (таблица переменных окружения и описание полей ChromeConfig).

## Контекст
- Модуль Chrome: `src/modules/chrome/` — сборка аргументов в `args.ts`, запуск в `launch.ts`, конфиг передаётся из `config`.
- Конфигурация: `src/modules/config/` — `ChromeConfig` в `types.ts`, чтение и валидация в `validate.ts`. Уже есть переменные `CHROME_PATH`, `CHROME_WINDOW_MODE`, `CHROME_USER_DATA_DIR` и т.д.
- Режимы окна (`windowMode`: kiosk, fullscreen, app, default) уже реализованы; размер и позиция при полноэкранных режимах могут быть неактуальны — в плане можно оговорить: добавлять `--window-size` и `--window-position` только для режимов, где окно не полноэкранное, или всегда при заданных переменных.

## Открытые вопросы
- Формат переменных для размера: две отдельные (`CHROME_WINDOW_WIDTH`, `CHROME_WINDOW_HEIGHT`) или одна (`CHROME_WINDOW_SIZE=1280x720`)?
- Формат переменных для позиции: две отдельные (`CHROME_WINDOW_POSITION_X`, `CHROME_WINDOW_POSITION_Y`) или одна (`CHROME_WINDOW_POSITION=100,200`)?
