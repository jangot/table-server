# Задача: Исправить запуск Chrome — размер контента и режим киоска

## Суть задачи
При запуске Chrome окно открывается в нормальном (ожидаемом) размере, но контент внутри браузера отображается значительно меньше, чем должен. Кроме того, возможна проблема с режимом киоска (`--kiosk`): системный интерфейс (панели, рамки окна) всё равно отображается, что говорит о том, что kiosk-режим не активируется корректно.

## Детали и требования
- Выяснить, почему контент отображается меньше ожидаемого при корректном размере окна (возможные причины: масштаб устройства `--force-device-scale-factor`, zoom, неправильный `window-size`)
- Проверить и исправить флаги запуска Chrome, отвечающие за kiosk-режим (`--kiosk` или `--app` + `--start-fullscreen`)
- Убедиться, что после запуска системный UI (рамки окна, панель задач) скрыт
- Не должно быть регрессий по уже работающим функциям: управление URL через CDP, переключение сцен и т.д.

## Контекст
- Задача 002 (`chrome-launch-and-control`) — базовый запуск Chrome
- Задача 017 (`chrome-window-size-env`) — управление размером окна через переменные окружения
- Запуск Chrome, вероятно, происходит через Puppeteer или прямой вызов процесса с набором флагов
- Проект запускается на Linux (возможно, без DE — Wayland/X11 в kiosk-сценарии)

## Открытые вопросы
- Какой именно флаг отвечает за kiosk в текущей конфигурации?
- Какой `--force-device-scale-factor` установлен сейчас (если вообще установлен)?
- Используется ли Puppeteer или прямой запуск Chrome?
- На каком дисплее/сервере запускается Chrome (X11, Wayland, Xvfb)?
